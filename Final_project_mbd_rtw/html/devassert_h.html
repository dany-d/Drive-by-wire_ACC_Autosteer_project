36/**



37   \page Error_detection_and_reporting Error detection and reporting



38



39   S32 SDK drivers can use a mechanism to validate data coming from upper software layers (application code) by performing



40   a number of checks on input parameters' range or other invariants that can be statically checked (not dependent on



41   runtime conditions). A failed validation is indicative of a software bug in application code, therefore it is important



42   to use this mechanism during development.



43



44   The validation is performed by using DEV_ASSERT macro.



45   A default implementation of this macro is provided in this file. However, application developers can provide their own



46   implementation in a custom file. This requires defining the CUSTOM_DEVASSERT symbol with the specific file name in the



47   project configuration (for example: -DCUSTOM_DEVASSERT="custom_devassert.h")



48



49   The default implementation accommodates two behaviors, based on DEV_ERROR_DETECT symbol:



50   - When DEV_ERROR_DETECT symbol is defined in the project configuration (for example: -DDEV_ERROR_DETECT), the validation



51   performed by the DEV_ASSERT macro is enabled, and a failed validation triggers a software breakpoint and further execution is



52   prevented (application spins in an infinite loop)



53   This configuration is recommended for development environments, as it prevents further execution and allows investigating



54   potential problems from the point of error detection.



55   - When DEV_ERROR_DETECT symbol is not defined, the DEV_ASSERT macro is implemented as no-op, therefore disabling all validations.



56   This configuration can be used to eliminate the overhead of development-time checks.



57



58   It is the application developer's responsibility to decide the error detection strategy for production code: one can opt to



59   disable development-time checking altogether (by not defining DEV_ERROR_DETECT symbol), or one can opt to keep the checks



60   in place and implement a recovery mechanism in case of a failed validation, by defining CUSTOM_DEVASSERT to point



61   to the file containing the custom implementation.



62 */

65/* If the CUSTOM_DEVASSERT symbol is defined, then add the custom implementation */
